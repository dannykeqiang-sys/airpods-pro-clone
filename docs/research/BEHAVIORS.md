## 1. 滚动触发动画 (Scroll-triggered Animations)

### 1.1 视差滚动 (Parallax Scrolling)
- **触发条件**：用户滚动到特定 Section 时
- **行为描述**：各 Section 的标题、图片、文字分层进入视口，产生视差效果
- **实现方式**：通过 IntersectionObserver 或 scroll 事件监听，控制元素的 transform 和 opacity
- **关键参数**：
  - 进入动画时长：约 600-800ms
  - 缓动函数：`cubic-bezier(0.25, 0.1, 0.25, 1)` 或自定义 spring 缓动
  - 触发阈值：元素进入视口约 30% 时开始动画

### 1.2 渐显动画 (Fade-in)
- **触发条件**：Section 进入视口
- **行为描述**：标题文字从下方渐显（translateY + opacity）
- **关键参数**：
  - 初始状态：`opacity: 0; transform: translateY(30px)`
  - 结束状态：`opacity: 1; transform: translateY(0)`
  - 时长：400-600ms

## 2. Hero 区域视频动画

### 2.1 Hero 视频播放
- **触发条件**：页面加载后自动播放
- **行为描述**：Hero 区域播放 AirPods Pro 3 展示视频，从 startframe 到 endframe 循环播放
- **关键参数**：
  - `autoplay: true`（实际通过 JS 控制）
  - `loop: true`
  - `muted: true`
  - `playsInline: true`
  - 视频分辨率：`large_2x.mp4`（约 2x 分辨率）

### 2.2 Hero 视频控制交互
- **触发条件**：用户 hover 或点击视频区域
- **行为描述**：视频播放器控制条在 hover 时渐显，包含播放/暂停、音量、全屏、PIP 等按钮
- **关键参数**：
  - 控制条渐显时长：200ms
  - 控制条背景：`linear-gradient(rgba(0, 0, 0, 0.24), rgba(0, 0, 0, 0.42), rgba(0, 0, 0, 0.56))`

## 3. Product Viewer 交互

### 3.1 产品细看 (Closer Look)
- **触发条件**：用户点击不同功能标签（音质、佩戴感、心率、防水、触控、充电盒）
- **行为描述**：产品图片切换，带有平滑过渡动画；每个标签对应不同的产品视角
- **关键参数**：
  - 切换动画：fade + scale 过渡
  - 时长：400ms
  - 缓动：`ease-in-out`

### 3.2 AR 空间查看
- **触发条件**：用户点击"在你的空间中查看"按钮
- **行为描述**：触发 AR Quick Look，在 3D 空间中查看 AirPods Pro

## 4. 噪声控制交互

### 4.1 噪声控制动画
- **触发条件**：滚动到噪声控制 Section
- **行为描述**：圆形音浪涟漪动画从中心向外扩散，展示"智能噪声控制"概念
- **关键参数**：
  - 动画类型：CSS keyframe + JS 控制
  - 涟漪扩散：scale 从 0 到 1，opacity 从 1 到 0
  - 时长：约 2-3 秒循环

### 4.2 噪声控制子功能切换
- **触发条件**：用户点击不同功能标签（降噪麦克风、语音突显、自适应音频、对话感知）
- **行为描述**：Gallery 图片切换，带有滑动过渡
- **关键参数**：
  - 切换方式：水平滑动或 fade
  - 时长：300-400ms

## 11. 导航交互

### 11.1 全局导航栏
- **触发条件**：hover 导航项
- **行为描述**：下拉菜单展开，显示子菜单
- **关键参数**：
  - 展开动画：height 从 0 到 auto，opacity 从 0 到 1
  - 时长：300ms
  - 缓动：`ease-in-out`

### 11.2 本地导航（子页面导航）
- **触发条件**：页面滚动
- **行为描述**：本地导航栏 sticky 定位，随滚动高亮当前 Section
- **关键参数**：
  - sticky 定位：`position: sticky; top: 0`
  - 高亮切换：颜色变化 `rgb(0, 113, 227)` → `rgb(110, 110, 115)`

## 14. 页面整体滚动行为

### 14.1 平滑滚动
- **行为描述**：页面使用 `scroll-behavior: smooth` 或 JS 控制的平滑滚动
- **关键参数**：
  - 导航锚点跳转：平滑滚动到对应 Section
  - 时长：约 500-800ms

### 14.2 Section 间距与节奏
- **行为描述**：各 Section 之间有充足的间距（约 80-160px），创造视觉呼吸感
- **关键参数**：
  - Section 间距：`padding-top/bottom: 80px ~ 160px`
  - Hero Section 与导航重叠：`section-welcome-localnav-overlap`

## 15. 响应式行为

### 15.1 桌面端 (≥1024px)
- 双列布局（图片 + 文字）
- 大字号标题
- 视频自动播放

### 15.2 移动端 (<768px)
- 单列布局
- 缩小字号
- 图片懒加载（`loading="lazy"`）
- 视频可能不自动播放

### 15.3 中等屏幕 (768px-1024px)
- 适度缩小的双列布局
- 中等字号
